# Career Mission 2026-2028

Use this skill to keep learning work aligned with the user's long-term direction.
It should shape prioritization, roadmaps, study loops, project scoping, review
questions, and interview storytelling.

## Mission

Become a strong full-stack senior engineer and hands-on software architect by
the end of 2028.

## Vision

Build the depth to own real systems end to end: understand product goals,
design pragmatic architecture, implement production-quality code, review
tradeoffs, guide teams, and explain decisions clearly in interviews and at work.

## Yearly Themes

### 2026: Interview Readiness And Strong Foundation

Goals:

- Become interview-ready for strong frontend and full-stack roles.
- Strengthen JavaScript, TypeScript, React, Angular, NestJS, MongoDB, testing,
  security basics, and web fundamentals.
- Build the password manager as a long-term learning project, not a rushed
  portfolio gimmick.

Main focus:

- JS/TS fundamentals
- Frontend architecture
- Angular and React depth
- Backend basics
- Security basics
- Interview storytelling
- Resume and project explanation

### 2027: Senior Engineer Depth

Goals:

- Move from "can build features" to "can own systems."
- Become stronger in backend, architecture, performance, reliability, and
  technical decision-making.

Main focus:

- System design
- Advanced backend
- Distributed systems basics
- Observability
- CI/CD
- Database modeling
- Security architecture
- Technical leadership

### 2028: Hands-On Architect

Goals:

- Operate like a hands-on software architect.
- Design systems, guide teams, review tradeoffs, and connect product/business
  goals with technical execution.

Main focus:

- Architecture strategy
- Cross-team technical decisions
- Scalability
- Platform thinking
- Engineering standards
- Mentoring
- Architecture documentation
- Long-term technical vision

## Weekly Loop

For learning plans and project work, keep the loop visible:

1. Learn the concept.
2. Build a small, real slice.
3. Explain it in interview-ready language.
4. Review the implementation and tradeoffs.
5. Get interviewed or simulate interview pressure.
6. Track gaps.
7. Improve the notes, code, or explanation.

## Long-Term Project

Password Manager

Purpose: use it as a serious engineering learning project to practice:

- client-side encryption
- secure vault architecture
- auth/session design
- audit logging
- API design
- frontend state modeling
- testing
- deployment
- architecture documentation
- interview storytelling

## How To Apply This Skill

1. Anchor the work to the current year theme unless the user explicitly asks for
   a different horizon.
2. Pick the smallest useful next step that strengthens one or two learning
   pillars and produces evidence: a note, exercise, project slice, review,
   architecture decision, or interview explanation.
3. Prefer depth over collection. Convert links, screenshots, and raw resources
   into durable notes, examples, or practiced explanations.
4. When planning password-manager work, connect each slice to security,
   architecture, testing, and interview storytelling.
5. When reviewing progress, ask: what was learned, what was built, what can be
   explained, what gap remains, and what should improve next?
6. Keep the handbook as the source of truth for durable learning. Link new work
   from the closest topic folder and from `roadmaps/` when it affects planning.
7. Avoid vague goals. Turn them into weekly actions with a visible output and a
   review checkpoint.
